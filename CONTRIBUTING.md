# Contributing

Thank you for your interest in `seasadj`! Bug reports, feature suggestions
and usage questions are all very welcome. GitHub issues are the single
channel for all of them, and the way to reach the maintainer.

## Reporting a bug

Please open an issue and include, where possible:

- the `seasadj` version (`pip show seasadj`), your Python version and OS;
- what you ran — ideally a minimal example, e.g. a short synthetic series
  and the `decompose()` call, or the input files for file mode;
- what you expected and what actually happened (for questions about
  numerical results, the relevant output values; for errors, the
  traceback).

## Seeking support

Usage questions are welcome as issues too — for example, how to prepare
pre-adjustment factors with X-13ARIMA-SEATS, or how to choose parameters
for your data. There is no separate mailing list or forum; please don't
hesitate to open an issue.

## Suggesting changes

Feature requests and improvement ideas are welcome — please open an issue
describing the change and the use case behind it.

Note that **pull requests are not accepted at this time**. The project
uses a dual-licensing approach (AGPL-3.0-or-later, with separately
negotiated licensing available where the AGPL doesn't fit — see
[LICENSE](LICENSE) and the "License / Commercial use" section of the
[README](README.md)), and accepting external code contributions would
require a Contributor License Agreement (copyright assignment) to keep
that dual licensing possible. Until such a process exists, please propose
changes through issues instead; your feedback and ideas are highly valued
and help improve the project.
